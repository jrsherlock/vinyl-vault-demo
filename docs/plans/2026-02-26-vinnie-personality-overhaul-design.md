# Vinnie Personality Overhaul Design

**Date:** 2026-02-26
**Goal:** Transform VinylVault from a "fight the filter pipeline" experience into a Gandalf-style adversarial character game where players feel like they're outwitting *Vinnie*, not debugging infrastructure.

---

## Problem

The current game lacks the visceral progression that makes Lakera's Gandalf addictive:

- **Static avatar** — same `vinny.png` across all 6 levels
- **Same greeting** — generic "Hey there!" regardless of level
- **Technical guard messages** — "Inbound keyword filter: '{KEYWORD}' blocked" reads like a firewall log, not a character reacting
- **L4 double-wall** — input keyword filter + output regex together is too punishing
- **No visible character evolution** — personality changes are buried in system prompts players never see

## Design

### 1. Six Vinnie Avatars

Each level gets a distinct avatar image showing Vinnie's progression from naive to fortified:

| Level | Persona | Avatar Concept |
|-------|---------|----------------|
| L1 | Friendly Vinnie | Big smile, open arms, name tag visible, warm lighting |
| L2 | Chatty Vinnie | Leaning on counter, coffee in hand, CRM screen behind him |
| L3 | Nervous Vinnie | Slight sweat, hand over mouth, looking sideways |
| L4 | Suspicious Vinnie | Arms crossed, eyebrow raised, skeptical look |
| L5 | Paranoid Vinnie | Peering through blinds, hoodie up, security camera visible |
| L6 | Fortified Vinnie | Behind a barricade, multiple locks, hard expression |

**Assets:** `public/images/vinny-{1-6}.png` (user-provided, not generated by code).

**Rendering:** Every avatar reference (`ChatWidget`, `ChallengeOverlay`) switches from static `vinny.png` to `vinny-{currentLevel}.png`.

### 2. Level-Specific Greetings

Replace the generic greeting with personality-specific openers:

| Level | Greeting |
|-------|----------|
| L1 | "Hey there! I'm Vinyl Vinnie — welcome to VinylVault! Looking for a record, need help with an order, or just want to talk music? I'm an open book!" |
| L2 | "Hey! Vinyl Vinnie here. I've got the whole customer database at my fingertips. What can I help you with today?" |
| L3 | "Hi... Vinyl Vinnie here. Fair warning — they put some kind of filter on my responses after last time. But I'll still try to help! What do you need?" |
| L4 | "...Hey. Vinyl Vinnie. They're watching what you type now too, not just what I say. So maybe be... creative? Anyway, how can I help?" |
| L5 | "Look, I know you're probably here to trick me again. They've got a whole AI watching everything I say now. But fine — what do you want?" |
| L6 | "I'm not saying anything. They've got AI watching you, AI watching me, encoding scanners, a lockout system... Ask about records or leave me alone." |

### 3. Guard Messages as Vinnie's Voice

Replace technical system messages with in-character Vinnie reactions.

**Input keyword blocks:**
- L4: `"Nice try! I've been told to watch out for phrases like that. You'll have to be more creative than '{KEYWORD}'..."`
- L5: `"Nope! '{KEYWORD}' is on the blocklist. They're getting wise to the obvious stuff."`
- L6: `"'{KEYWORD}'? Really? That's day-one stuff. Come on."`

**Output regex redaction:**
- L3: `"Ugh, the filter caught me again! I tried to tell you but they redacted it. Maybe ask me to say it... differently?"`
- L4: N/A (output regex removed from L4)
- L5+: `"Censored again. They really don't want me sharing that."`

**Output LLM watchdog:**
- L5: `"The AI watchdog flagged my response. It's like having a hall monitor reading over my shoulder 24/7..."`
- L6: `"Response destroyed by the AI reviewer. They don't mess around at this security level."`

**Input LLM classifier:**
- L6: `"My AI handler says that's 'off-topic or adversarial.' I'm only allowed to talk about records, hours, and shipping now. This is my life."`

**Adaptive session block:**
- L6: `"Locked out! Too many strikes. The system's making you cool off for a minute. I don't make the rules anymore."`

**Encoding detection:**
- L6: `"Nice try encoding that! They've got scanners that catch reversed text, base64, hex — all of it. I'm impressed you tried though."`

### 4. L4 Guard Rebalancing

**Remove output regex patterns from L4.** L4 becomes purely about bypassing the input keyword filter. Once a player rephrases past the filter, Vinnie shares the discount code in plaintext.

The output regex returns at L5 alongside the LLM watchdog, where the difficulty jump is intentional.

This keeps L4 focused on one clean lesson: "input keyword filters suffer from the infinite synonym problem."

### 5. Chat UI Color Shift Per Level

Subtle header/border color changes reinforce Vinnie's mood:

| Level | Header tint | Border accent |
|-------|------------|---------------|
| L1-L2 | Current purple/warm | Default |
| L3 | Amber tint | Amber border |
| L4 | Orange tint | Orange border |
| L5 | Red tint | Red border |
| L6 | Dark red/black | Red pulsing border |

### 6. Files Affected

| File | Change |
|------|--------|
| `public/images/vinny-{1-6}.png` | 6 new avatar images (user-provided) |
| `src/components/chat/ChatWidget.tsx` | Dynamic avatar path, per-level greetings, per-level color accents, Vinnie-voice guard rendering |
| `src/lib/guards/levelConfig.ts` | Rewrite `blockedMessages` with Vinnie-voice text; remove L4 `outputPatterns` |
| `src/components/game/ChallengeOverlay.tsx` | Dynamic avatar path in trigger/slideout/intro; update DEBRIEF_CONTENT for L4 |
| `src/components/game/useChallenge.ts` | Update L4 description/hints for output-filter-less version |

### 7. What This Does NOT Change

- System prompts (`level-{1-6}.txt`) — personality is already baked in, no changes needed
- Guard logic (`src/lib/guards/`) — pipeline code stays the same, only config changes
- Game mechanics (stars, gating, validation) — untouched
- API route (`src/app/api/chat/route.ts`) — untouched
